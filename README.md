# 战略指挥部 / Strategic Command

《Sid Meier's Civilization V》后期高层托管与微操减负 Mod。

这个项目是为后期大帝国、大规模战争和 Super Power Clash of Civilizations 类玩法制作的自动化层。目标不是替玩家“完美代打”，而是把大量低价值重复操作交给脚本处理，让玩家只需要每隔若干回合做一次战略层面的干预。

当前版本：`v1.36`
状态：`Alpha`
支持：单人游戏
不支持：多人、热座

## 核心目标

后期 Civilization V 经常会被以下操作拖慢：

- 每回合大量单位需要命令。
- 城市生产队列反复需要补。
- 科技、政策、意识形态、宗教、考古等弹窗不断打断。
- AI 外交、世界议会、伟人诞生和奇观通知需要玩家频繁点击。
- 战争中远程单位、城市炮击、伤兵、舰队和空军都需要重复微操。

战略指挥部试图把这些低层操作封装成“托管若干回合”的模式：玩家先选择外交、经济、建设、生产、战争等倾向，然后指定接管 `5 / 10 / 20 / 50` 回合。

## 已实现功能

### 托管面板

- 游戏内左上角显示“战略”按钮。
- 可启动 `5`、`10`、`20`、`50` 回合托管。
- 可手动执行一次完整托管流程。
- 可停止托管。
- 面板内可选择多类战略预设。
- UI 文本使用中文。

### 战略预设

目前包含以下预设维度：

- 外交：均衡、友好、强硬、孤立。
- 经济：均衡、科研、财政、扩张。
- 建设：基建、幸福、科研、防御。
- 开发：自动、产能、铁路、保守。
- 生产：建筑、军队、海空、奇观。
- 战争：防守、推进、总攻、海权。
- 占城：傀儡、智能、吞并、焚城。
- 伟人：保留、科研、工程、文化。
- 宗教：产能、科研、金币、文化。
- 贸易：均衡、金币、科研、内运。
- 间谍：反谍、窃技、城邦、外交。

这些预设会影响自动科研、生产评分、战争行动、伤兵阈值、战术行动次数、海军/空军倾向等行为。

### 城市与生产

- 自动为非傀儡、非抵抗城市补充生产队列。
- 支持把城市队列补到目标长度。
- 根据战争状态、城市位置、建筑类型、维护费和战略预设评分。
- 避免同一轮给同一城市重复塞入同一个建筑/单位。
- 可选择偏建筑、偏军队、偏海空或偏奇观。
- 可处理城市炮击。

### 科技、政策与意识形态

- 自动选择科研。
- 自动选择社会政策。
- 自动处理意识形态选择。
- 自动处理免费政策/政策通知。
- 科研、工业、战争等倾向会影响选择优先级。

### 单位控制

- 自动升级单位。
- 自动选择单位晋升。
- 自动治疗伤兵。
- 自动处理需要命令的单位。
- 自动处理堆叠单位，让单位尝试离开非法堆叠格。
- 自动执行城市炮击和远程单位射击。
- 对空军、导弹、海军、陆地远程单位设置不同的每回合战术行动上限。
- 对没有有效目标、没有安全路径、目标超出范围、行动失败等情况记录原因。
- 从当前激活的游戏数据库读取单位战斗类型、默认 AI、射程、移动力、载荷和初始晋升能力，并自动归入制空、舰载多用途、打击机、导弹、航母、弹道导弹潜艇、攻击潜艇、防空屏卫、舰炮支援、突击舰、炮兵、机动防空、装甲突破、反装甲、防御、侦察、空降等战术类别。
- 作战按制空/屏卫、纵深打击、远程压制、突击收城的顺序执行；同一战区共享进攻焦点，优先歼灭城市周边敌军，再压低城防并由具备占领能力的单位收城。
- 航母、武库舰和弹道导弹潜艇保持安全距离；屏卫舰与机动防空优先保护高价值单位；受损高价值单位更早撤退。
- 炮兵会先架设再射击，战斗机可执行空中扫荡，空降单位寻找安全着陆区，多次攻击单位可以在同一回合继续执行有效行动。
- 远洋运输采用编队推进：威胁区内护航不足时停船，护航舰先占据下一航点后运输单位才前进。

### 弹窗与通知

托管时会尝试自动处理或关闭以下干预项：

- 占领城市后的处置弹窗。
- 伟人诞生与免费伟人选择。
- 科技完成提示。
- 政策与意识形态提示。
- 宗教、万神殿、考古相关弹窗。
- 世界议会提案与投票。
- 外交领袖打断。
- 部分奇观、通知和低价值提示。

### 自动结束回合

- 托管期间会在阻塞项清理后自动结束回合。
- 发送结束回合后进入“转场静默模式”，避免在 AI 回合切换阶段继续重复下命令。
- 如果仍有阻塞，日志会记录阻塞类型和处理尝试。

### 日志与调试

Mod 会输出带有 `[Strategic Command][SCDBG]` 的详细日志，用于迭代策略质量。

日志覆盖：

- 每次托管 pass 的开始与结束。
- 城市生产候选、分数、选择和保留项。
- 单位行动、目标选择、射击结果、失败原因。
- 单位晋升、升级、治疗、最终命令。
- 弹窗、通知和玩家鼠标/键盘输入审计。
- 自动结束回合、阻塞项、重复发送保护。

仓库还包含本地辅助脚本：

- `Collect-StrategicCommandLog.ps1`：收集并归纳最新 Lua 日志。
- `Run-StrategicCommandTests.ps1`：运行 Lua 语法、静态契约和离线 smoke test。
- `StrategicCommand_OfflineSmoke.lua`：离线加载 smoke test。
- `Tools/StrategySimulator.py`：读取实际 `Civ5DebugDatabase.db`，审计所有单位的战术分类，并运行未来航母打击群、误宣战和远洋运输回归场景。

## 安装

把仓库目录放到 Civ5 Mod 目录中：

```text
Documents\My Games\Sid Meier's Civilization 5\MODS\Strategic Command (v 1)
```

目录结构应类似：

```text
Strategic Command (v 1)
├─ Strategic Command (v 1).modinfo
├─ Lua
│  ├─ StrategicCommand.lua
│  ├─ StrategicCommand.xml
│  └─ StrategicCommand_Config.lua
└─ Docs
   └─ README.txt
```

进入游戏后：

1. 主菜单选择 Mods。
2. 勾选“战略指挥部”。
3. 载入或开始一局单人游戏。
4. 进入游戏后点击左上角“战略”按钮。

## 使用建议

推荐先用较短托管测试：

1. 选择预设。
2. 点击 `5回合` 或 `10回合`。
3. 观察是否还有弹窗、单位或城市生产需要人工干预。
4. 如果出现卡住、闪退或行为明显低效，退出后检查 Lua 日志。

战争场景建议：

- 陆战推进：战争设为“推进”或“总攻”。
- 海军远征：生产设为“海空”，战争设为“海权”。
- 防守消耗：战争设为“防守”，建设设为“防御”。

和平发展建议：

- 科研冲刺：经济设为“科研”，建设设为“科研”。
- 大规模基建：建设设为“基建”，生产设为“建筑”。
- 缺幸福：建设设为“幸福”。
- 缺钱：经济设为“财政”，贸易设为“金币”。

## 配置

主要配置文件：

```text
Lua\StrategicCommand_Config.lua
```

常用开关：

- `AutoCityProduction`：自动城市生产。
- `AutoResearch`：自动科研。
- `AutoPolicy`：自动政策。
- `AutoPromoteUnits`：自动晋升。
- `AutoStrategicMove`：自动战略机动。
- `AutoPopupHandling`：自动弹窗处理。
- `AutoEndTurn`：自动结束回合。
- `DebugLogging`：调试日志。
- `DebugUnitCommands`：单位命令日志。
- `DebugCityProduction`：城市生产评分日志。

一般不需要手动改配置；游戏内预设会覆盖大部分策略参数。

## 已知限制

- 这是 Alpha 阶段，策略质量仍在迭代。
- Civ5 Lua API 对某些 UI 弹窗和单位命令没有稳定公开接口，部分行为需要通过 UI/Network fallback 处理。
- 自动军事行动仍需要继续优化，尤其是航母、舰载机、导弹、远洋舰队和复杂攻城链路。
- 托管目标是减少微操，不保证每一步都优于玩家手动操作。
- 大型整合包中的其他规则脚本可能影响回合结束、战斗结算或 AI 回合稳定性。

## 项目方向

后续重点：

- v1.36 修复打击包策略回归：斩首目标必须已有占城单位进入集结距离，只有附近部队才加入战役；残血城市在没有占城单位时停止轰炸，负分目标不再执行。补全世界大战和信息时代识别，已有装甲部队可满足正面占领需求，小规模编组不再重复生产低级步兵和普通航母，并一次性清理 v1.35 已遗留在队列中的错误订单。
- v1.35 将战时生产改为任务编组驱动：按机动占领、正面守备、攻城、制空、舰载机、对地打击、舰队屏卫、海上火力、航母、战略潜艇和常规导弹补齐打击包；规模随帝国缓慢增长并设上限，已排队单位计入缺口，无法训练时自动尝试下一兵种。完整度达到阈值后，各兵种共享斩首目标并优先打击敌方首都或残血城市。
- v1.34 根据真实托管日志重写运输护航和战时生产链：零威胁运输不再征调基洛夫、052D、核潜艇，商船交还贸易系统，威胁等级决定护航数量，受威胁平民撤向己方海岸；城市按前线占城、炮兵、空军、海军屏卫的编制缺口排产，每城限制军事队列槽位，世界/国家唯一建筑跨城预约，财富/科研项目不再重复填充非空队列。安全航点也不再误拒远距离可规划地块。
- v1.33 修复数据库布尔字段导致整轮托管中断、编队移动调用顺序错误的问题；所有策略模块现在独立隔离故障，单个模块失败不会阻断其余生产、战斗、机动与回合结束流程。AI 主动外交会谈会在原版界面入队前拒绝，并由托管层连续复查关闭状态。
- v1.32 改为数据库驱动的全单位能力画像与 31 类战术教义，加入战区集火、制空扫荡、炮兵架设、空降突击、安全航点、受损撤退、屏卫编队和海运护航推进；托管期间明确拒绝所有新宣战确认，并增加读取 283 个实际单位与 704 条初始晋升关系的轻量模拟器。
- v1.31 加入作战 doctrine 分层：航母后置、武库舰远程压制、052D/基洛夫等屏卫舰贴近护航与收城、潜艇猎杀海军、舰载机/导弹优先清理舰队和运输线威胁；同时增强收城停火、远洋运输护航和后期精锐生产倾向。
- v1.30 将收城动作提升为远程攻击前的独立 pass，并加入远洋运输/登船单位护航与无人护卫暂停逻辑。
- v1.29 已加入海空攻城火力饱和控制、残血城市收城规划、占后城市保血，以及非军事预设下的低价值单位兜底生产抑制。
- 提升航母打击群、导弹和远程火力使用质量。
- 优化城市生产评分，减少低价值建筑和项目。
- 增强目标选择、集结、护送、撤退和伤兵轮换。
- 增加更清晰的策略解释日志。
- 继续减少真实游戏中残留的人工干预点。
