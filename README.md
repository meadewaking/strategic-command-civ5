# 战略指挥部 / Strategic Command

《Sid Meier's Civilization V》后期高层托管与微操减负 Mod。

这个项目是为后期大帝国、大规模战争和 Super Power Clash of Civilizations 类玩法制作的自动化层。目标不是替玩家“完美代打”，而是把大量低价值重复操作交给脚本处理，让玩家只需要每隔若干回合做一次战略层面的干预。

当前版本：`v1.24`
状态：`Alpha`
支持：单人游戏
不支持：多人、热座

## 核心目标

后期 Civilization V 经常会被以下操作拖慢：

- 每回合大量单位需要命令。
- 城市生产队列反复需要补。
- 科技、政策、意识形态、宗教、考古等弹窗不断打断。
- AI 外交、世界议会、伟人诞生和奇观通知需要玩家频繁点击。
- 战争中远程单位、城市炮击、伤兵、舰队和空军都需要重复微操。

战略指挥部试图把这些低层操作封装成“托管若干回合”的模式：玩家先选择外交、经济、建设、生产、战争等倾向，然后指定接管 `5 / 10 / 20 / 50` 回合。

## 已实现功能

### 托管面板

- 游戏内左上角显示“战略”按钮。
- 可启动 `5`、`10`、`20`、`50` 回合托管。
- 可手动执行一次完整托管流程。
- 可停止托管。
- 面板内可选择多类战略预设。
- UI 文本使用中文。

### 战略预设

目前包含以下预设维度：

- 外交：均衡、友好、强硬、孤立。
- 经济：均衡、科研、财政、扩张。
- 建设：基建、幸福、科研、防御。
- 开发：自动、产能、铁路、保守。
- 生产：建筑、军队、海空、奇观。
- 战争：防守、推进、总攻、海权。
- 占城：傀儡、智能、吞并、焚城。
- 伟人：保留、科研、工程、文化。
- 宗教：产能、科研、金币、文化。
- 贸易：均衡、金币、科研、内运。
- 间谍：反谍、窃技、城邦、外交。

这些预设会影响自动科研、生产评分、战争行动、伤兵阈值、战术行动次数、海军/空军倾向等行为。

### 城市与生产

- 自动为非傀儡、非抵抗城市补充生产队列。
- 支持把城市队列补到目标长度。
- 根据战争状态、城市位置、建筑类型、维护费和战略预设评分。
- 避免同一轮给同一城市重复塞入同一个建筑/单位。
- 可选择偏建筑、偏军队、偏海空或偏奇观。
- 可处理城市炮击。

### 科技、政策与意识形态

- 自动选择科研。
- 自动选择社会政策。
- 自动处理意识形态选择。
- 自动处理免费政策/政策通知。
- 科研、工业、战争等倾向会影响选择优先级。

### 单位控制

- 自动升级单位。
- 自动选择单位晋升。
- 自动治疗伤兵。
- 自动处理需要命令的单位。
- 自动处理堆叠单位，让单位尝试离开非法堆叠格。
- 自动执行城市炮击和远程单位射击。
- 对空军、导弹、海军、陆地远程单位设置不同的每回合战术行动上限。
- 对没有有效目标、没有安全路径、目标超出范围、行动失败等情况记录原因。

### 弹窗与通知

托管时会尝试自动处理或关闭以下干预项：

- 占领城市后的处置弹窗。
- 伟人诞生与免费伟人选择。
- 科技完成提示。
- 政策与意识形态提示。
- 宗教、万神殿、考古相关弹窗。
- 世界议会提案与投票。
- 外交领袖打断。
- 部分奇观、通知和低价值提示。

### 自动结束回合

- 托管期间会在阻塞项清理后自动结束回合。
- 发送结束回合后进入“转场静默模式”，避免在 AI 回合切换阶段继续重复下命令。
- 如果仍有阻塞，日志会记录阻塞类型和处理尝试。

### 日志与调试

Mod 会输出带有 `[Strategic Command][SCDBG]` 的详细日志，用于迭代策略质量。

日志覆盖：

- 每次托管 pass 的开始与结束。
- 城市生产候选、分数、选择和保留项。
- 单位行动、目标选择、射击结果、失败原因。
- 单位晋升、升级、治疗、最终命令。
- 弹窗、通知和玩家鼠标/键盘输入审计。
- 自动结束回合、阻塞项、重复发送保护。

仓库还包含本地辅助脚本：

- `Collect-StrategicCommandLog.ps1`：收集并归纳最新 Lua 日志。
- `Run-StrategicCommandTests.ps1`：运行 Lua 语法、静态契约和离线 smoke test。
- `StrategicCommand_OfflineSmoke.lua`：离线加载 smoke test。

## 安装

把仓库目录放到 Civ5 Mod 目录中：

```text
Documents\My Games\Sid Meier's Civilization 5\MODS\Strategic Command (v 1)
```

目录结构应类似：

```text
Strategic Command (v 1)
├─ Strategic Command (v 1).modinfo
├─ Lua
│  ├─ StrategicCommand.lua
│  ├─ StrategicCommand.xml
│  └─ StrategicCommand_Config.lua
└─ Docs
   └─ README.txt
```

进入游戏后：

1. 主菜单选择 Mods。
2. 勾选“战略指挥部”。
3. 载入或开始一局单人游戏。
4. 进入游戏后点击左上角“战略”按钮。

## 使用建议

推荐先用较短托管测试：

1. 选择预设。
2. 点击 `5回合` 或 `10回合`。
3. 观察是否还有弹窗、单位或城市生产需要人工干预。
4. 如果出现卡住、闪退或行为明显低效，退出后检查 Lua 日志。

战争场景建议：

- 陆战推进：战争设为“推进”或“总攻”。
- 海军远征：生产设为“海空”，战争设为“海权”。
- 防守消耗：战争设为“防守”，建设设为“防御”。

和平发展建议：

- 科研冲刺：经济设为“科研”，建设设为“科研”。
- 大规模基建：建设设为“基建”，生产设为“建筑”。
- 缺幸福：建设设为“幸福”。
- 缺钱：经济设为“财政”，贸易设为“金币”。

## 配置

主要配置文件：

```text
Lua\StrategicCommand_Config.lua
```

常用开关：

- `AutoCityProduction`：自动城市生产。
- `AutoResearch`：自动科研。
- `AutoPolicy`：自动政策。
- `AutoPromoteUnits`：自动晋升。
- `AutoStrategicMove`：自动战略机动。
- `AutoPopupHandling`：自动弹窗处理。
- `AutoEndTurn`：自动结束回合。
- `DebugLogging`：调试日志。
- `DebugUnitCommands`：单位命令日志。
- `DebugCityProduction`：城市生产评分日志。

一般不需要手动改配置；游戏内预设会覆盖大部分策略参数。

## 已知限制

- 这是 Alpha 阶段，策略质量仍在迭代。
- Civ5 Lua API 对某些 UI 弹窗和单位命令没有稳定公开接口，部分行为需要通过 UI/Network fallback 处理。
- 自动军事行动仍需要继续优化，尤其是航母、舰载机、导弹、远洋舰队和复杂攻城链路。
- 托管目标是减少微操，不保证每一步都优于玩家手动操作。
- 大型整合包中的其他规则脚本可能影响回合结束、战斗结算或 AI 回合稳定性。

## 项目方向

后续重点：

- 提升航母打击群、导弹和远程火力使用质量。
- 优化城市生产评分，减少低价值建筑和项目。
- 增强目标选择、集结、护送、撤退和伤兵轮换。
- 增加更清晰的策略解释日志。
- 继续减少真实游戏中残留的人工干预点。
